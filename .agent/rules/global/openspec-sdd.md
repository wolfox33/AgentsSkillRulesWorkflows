# OpenSpec SDD-First Rules

- OpenSpec e a fonte de verdade para requisitos, escopo, contratos e criterios de aceitacao.
- Agentes devem consumir a spec aprovada e executar implementacao/validacao.
- Nao recriar roadmap, backlog de produto, contratos de API ou criterios globais fora do OpenSpec.
- Quando houver lacuna ou ambiguidade, abrir `spec-questions.md` e solicitar ajuste de spec.
- Mudancas de escopo/regra de negocio exigem update no OpenSpec antes da implementacao.
